Sitemap

Technical Description

HTML Structure:

  • I made sure to organize the content with clear headings and paragraphs so everything feels easy to read and navigate.
  • I used headings to break up the content, making it simple for users to skim and find what they're looking for quickly.
  • To help users navigate around the site, I added links to jump to different parts, which makes it more interactive and user-friendly.

CSS Styling:

  • I picked fonts that are easy on the eyes, with a clean and modern look, to keep the text readable and the overall design sleek.
  • For the layout, I spaced out the elements thoughtfully so nothing feels cramped. It keeps everything balanced and neat.
  • To avoid a cluttered look, I gave sections a bit of breathing room and used visual separation like different background colors to keep things clear and distinct.

Accessibility

  • Clear Headings:I used clear and structured headings (h1, h2, h3) to help screen readers understand the content hierarchy and navigate the page easily.
  • Navigation Links:All links are clearly visible and accessible, ensuring that users can easily navigate between pages
  • Hover Effects & Focus Styles:To enhance navigation for keyboard users, I added focus and hover styles on clickable elements. This helps users track where they are when navigating via keyboard or mouse.

    User Profile

    Technical Description

    HTML structure

    • The page is well-organized with distinct sections that include a navigation bar, a user profile sidebar, and multiple content areas. The content is broken up into manageable chunks, such as bio, skills, goals, and frustrations. This makes it easy to read and navigate.
    • The layout uses a mix of flexbox and grid systems to organize elements cleanly. The sidebar holds the user's profile and skills, while the main content area is used for bio, goals, frustrations, and contact info. The design adapts well to different screen sizes, so the page looks great on both desktops and mobile devices.
    • There’s a focus on clear, easy-to-read fonts, with elements like profile images and buttons designed to be visually engaging. Colors are used effectively to separate sections and add contrast, making everything easy to spot and interact with.
    • Buttons and links throughout the page have smooth hover effects, creating a modern feel. The calendar has special color-coded days to indicate availability, and the form inputs are designed to be user-friendly with visual feedback when selected.

    Accessibility

  • Navigation:The navigation is simple and clear, with easy-to-follow links and dropdown menus. This makes it accessible to people using screen readers or keyboard navigation.
  • Readable structure:The page uses proper HTML structure, making it easier for screen readers to interpret the content. It’s organized in a way that allows assistive technologies to quickly understand the layout.
  • Color Contrast:Good color contrast ensures text stands out against the background, making it more readable for people with visual impairments.
  • Responsive Design:The page adjusts to different screen sizes, ensuring that users can comfortably view the content on desktops, tablets, and smartphones.

    Content Page

    Technical Description

    HTML structure

    • It includes links to various sections like Home, Volunteer, Feedback, Table, and others. These links are organized with a dropdown menu for "Contents" and "Validation" categories.
    • Each section has relevant paragraphs with facts, and explanations
    • The footer is styled with a gradient background, a blur effect, and a container that centers the footer links.
    • A clean and simple design is maintained throughout the page. The layout uses responsive design principles to ensure the page adapts to different screen sizes.

    Accessibility

  • Buttons -The "Go to Top" button is clearly visible and clickable. The hover effect adds interactivity, improving the user's experience.
  • Readable Fonts -The fonts used in the body are large enough for most users to read easily, and the line height is appropriate to avoid crowding the text.
  • Flexibility -The layout adjusts for different screen sizes, ensuring that content is not cut off or difficult to interact with on smaller devices.

    Challenges and Lessons Learned

  • Balancing Interactivity and AccessibilityMaking the sitemap keyboard-navigable while maintaining smooth hover effects was challenging.
  • Scalability and ResponsivenessThe SVG had to be proportional across different screen sizes (desktop, tablet, mobile). Adjusting viewBox settings helped achieve this.
  • Ensuring Readability in SVG ElementsSome text positioning needed fine-tuning to ensure visibility within nodes while maintaining clear UI structure.
